DOM Manipulation hiệu quả: Lời khuyên và thủ thuật

Khi làm việc với JavaScript, việc thao tác với DOM (Document Object Model) là một trong những nhiệm vụ phổ biến nhất. Trong bài viết này, chúng ta sẽ tìm hiểu các cách truy cập và thay đổi DOM hiệu quả.

1. DOM là gì?

DOM (Document Object Model) là một cấu trúc dữ liệu biểu diễn trang web dưới dạng một cây các node. Mỗi node có thể là một phần tử HTML, văn bản, hoặc thuộc tính. DOM cung cấp một cách để truy cập và thao tác với các phần tử HTML trong trang web.

Cấu trúc DOM

  • Document: Node gốc, đại diện cho toàn bộ trang web
  • Element nodes: Các thẻ HTML như div, p, span
  • Text nodes: Nội dung văn bản trong các element
  • Attribute nodes: Các thuộc tính của element
document
        └── html
            ├── head
            │   ├── title
            │   └── meta
            └── body
                ├── div
                │   ├── h1
                │   └── p
                └── span

DOM Manipulation bao gồm:

  • Truy cập elements: Tìm và chọn các phần tử trong DOM
  • Thay đổi nội dung: Sửa đổi text hoặc HTML của elements
  • Thay đổi styles: Điều chỉnh CSS của elements
  • Thêm/xóa elements: Tạo mới hoặc xóa elements khỏi DOM
  • Xử lý sự kiện: Phản hồi với các tương tác của người dùng

2. Cách truy cập và thay đổi DOM hiệu quả

Truy cập DOM

Truy cập bằng ID

let element = document.getElementById('myElement');
  • document.getElementById() là một trong những cách truy cập nhanh nhất vì trình duyệt tối ưu hóa việc tìm kiếm theo ID.
  • Nếu bạn biết chính xác ID của phần tử, sử dụng phương thức này sẽ nhanh hơn các phương thức khác như querySelector()..

Truy cập bằng class hoặc tag name

let elements = document.getElementsByClassName('myClass');
        Array.from(elements).forEach((element) => {
            console.log(element);
        });
  • Nếu bạn cần thao tác với nhiều phần tử cùng lúc, hãy sử dụng document.getElementsByClassName() hoặc document.getElementsByTagName() để lấy một danh sách phần tử.
  • Tuy nhiên, lưu ý rằng các phương thức này trả về HTMLCollection, không phải NodeList, do đó bạn sẽ cần chuyển đổi nếu muốn sử dụng các phương thức như forEach().

Truy cập với querySelector

let element = document.querySelector('.myClass');
  • Phương thức querySelector() và querySelectorAll() cung cấp một cách tiếp cận linh hoạt hơn để truy cập các phần tử thông qua CSS selectors.
  • Tuy nhiên, chúng có thể chậm hơn một chút so với getElementById() khi bạn làm việc với ID vì phải xử lý nhiều loại selector khác nhau.

Thay đổi DOM

Thêm hoặc sửa nội dung của phần tử

element.textContent = "New content here!";
  • Thay vì sử dụng innerHTML, hãy sử dụng textContent hoặc innerText khi bạn chỉ cần thay đổi văn bản của phần tử.
  • innerHTML sẽ thực thi mã HTML, có thể gây nguy hiểm nếu dữ liệu không được xác minh.

Thêm hoặc sửa thuộc tính

element.setAttribute('src', 'new-image.jpg');
  • Sử dụng setAttribute() hoặc các thuộc tính trực tiếp của phần tử để thay đổi thuộc tính.
  • Điều này giúp dễ dàng chỉnh sửa thuộc tính như src, href, alt, vv....

Thêm phần tử mới

let newElement = document.createElement('div');
newElement.textContent = 'New element';
parentElement.appendChild(newElement);
  • Sử dụng appendChild() hoặc insertBefore() để thêm phần tử vào DOM mà không phải tái tạo lại toàn bộ cấu trúc.

4. Các tối ưu hiệu suất

Sử dụng DocumentFragment

let fragment = document.createDocumentFragment();
        fragment.appendChild(child1);
        fragment.appendChild(child2);
        parentElement.appendChild(fragment);
  • Giảm số lần thao tác với DOM
  • Cải thiện hiệu suất khi thêm nhiều phần tử

Sử dụng requestAnimationFrame()

  • Khi bạn cần thực hiện các thay đổi liên quan đến animation, requestAnimationFrame() giúp đồng bộ hóa các thao tác và cải thiện hiệu suất.

Tránh sử dụng innerHTML khi không cần thiết

  • innerHTML có thể làm cho trình duyệt phải tái tạo lại DOM, điều này gây ra sự giảm hiệu suất.
  • Hãy sử dụng textContent hoặc innerText thay cho trường hợp chỉ cần thay đổi văn bản.

Debouncing Example

function debounce(func, wait) {
            let timeout;
            return function executedFunction(...args) {
                const later = () => {
                    clearTimeout(timeout);
                    func(...args);
                };
                clearTimeout(timeout);
                timeout = setTimeout(later, wait);
            };
        }
  • Giảm số lần gọi hàm trong các sự kiện liên tục
  • Cải thiện hiệu suất cho các sự kiện như scroll, resize

Kết luận

Việc thao tác DOM hiệu quả không chỉ giúp cải thiện hiệu suất mà còn làm cho ứng dụng của bạn trở nên mượt mà và dễ dàng bảo trì hơn. Hãy luôn nhớ tối ưu hóa cách bạn thao tác với DOM để đảm bảo rằng ứng dụng của bạn hoạt động nhanh và hiệu quả nhất có thể.
